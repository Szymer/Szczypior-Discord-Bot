# Szczypior Discord Bot Repository

## File Structure

### Root Directory
- **docker-compose.yml**: Configuration file for Docker Compose, defining services and their dependencies.
- **docs/**: Contains documentation files for various aspects of the project, such as API, architecture, database migrations, and deployment.
- **htmlcov/**: Directory for HTML coverage reports generated by testing tools.
- **infrastructure/**: Contains Dockerfiles and other infrastructure-related files for services like the database, Redis, and Django.
- **libs/**: Shared libraries and modules used across multiple services.
- **services/**: Contains the main services of the project, including the Discord bot, database service, and LLM service.
- **tests/**: Test files for unit and integration testing.
- **tools/**: Utility scripts for deployment, testing, and debugging.

### Key Directories and Files

#### `docs/`
- **api.md**: Documentation for the API endpoints.
- **architecture.md**: Overview of the system architecture.
- **context.md**: Context and background information about the project.
- **db.md**: Details about the database schema and migrations.
- **deployment.md**: Instructions for deploying the project.
- **EXAMPLES.md**: Example use cases and scenarios.
- **postgres_migration.md**: Documentation for PostgreSQL migrations.
- **refaktor.md**: Notes on refactoring efforts.

#### `infrastructure/`
- **docker/**: Contains Dockerfiles for various services.
  - **db-service.Dockerfile**: Dockerfile for the database service.
  - **discord-bot.Dockerfile**: Dockerfile for the Discord bot.
  - **django.Dockerfile**: Dockerfile for the Django service.
  - **llm.Dockerfile**: Dockerfile for the LLM service.
- **postgres/**: Contains PostgreSQL initialization scripts and migrations.
- **redis/**: Placeholder for Redis-related configurations.

#### `libs/`
- **shared/**: Shared modules and utilities.
  - **config.py**: Configuration management.
  - **constants.py**: Project-wide constants.
  - **exceptions.py**: Custom exception classes.
  - **schemas/**: Data schemas for conversations, messages, and users.

#### `services/`
- **db-service/**: Database service implementation.
  - **requirements.txt**: Python dependencies for the database service.
  - **app/**: Source code for the database service.
- **discord-bot/**: Discord bot implementation.
  - **config.json**: Configuration file for the bot.
  - **Dockerfile**: Dockerfile for the bot.
  - **fly.toml**: Configuration for Fly.io deployment.
  - **requirements.txt**: Python dependencies for the bot.
  - **bot/**: Source code for the bot.
- **llm-service/**: LLM service implementation.
  - **requirements.txt**: Python dependencies for the LLM service.
  - **app/**: Source code for the LLM service.

#### `tests/`
- **discord-bot/**: Unit tests for the Discord bot.
- **integration/**: Integration tests for the project.
- **llm-service/**: Unit tests for the LLM service.

#### `tools/`
- **deploy_schema.py**: Script for deploying database schemas.
- **show_user_info.py**: Script for displaying user information.
- **test_sheets_connection.py**: Script for testing Google Sheets connections.
- **test_time_extraction.py**: Script for testing time extraction logic.

## Logic Overview

### Discord Bot
The Discord bot is implemented in the `services/discord-bot/` directory. It uses the `discord.py` library to interact with the Discord API. The bot's main responsibilities include:
- Responding to user commands.
- Managing conversations and interactions.
- Integrating with other services like the database and LLM.

### Database Service
The database service, located in `services/db-service/`, handles data storage and retrieval. It uses PostgreSQL as the database backend and provides APIs for:
- User management.
- Conversation history.
- Message storage.

### LLM Service
The LLM service, found in `services/llm-service/`, integrates with large language models to provide advanced conversational capabilities. It communicates with the Discord bot and database service to:
- Generate responses.
- Analyze user input.
- Perform natural language processing tasks.

### Infrastructure
The `infrastructure/` directory contains Dockerfiles and configurations for deploying the services. Docker Compose is used to orchestrate the services, ensuring seamless integration and scalability.

### Testing
The `tests/` directory includes unit and integration tests to ensure the reliability and correctness of the codebase. Coverage reports are generated and stored in the `htmlcov/` directory.

### Deployment
Deployment is managed using Fly.io, with configurations defined in `services/discord-bot/fly.toml`. The `Deploy to Fly.io` task automates the deployment process.
